# 章节主张、证据与推演模式对齐

本文件服务于 `/UPTW-plan`，用于把长篇毕业论文的章节写作目标落到“可写”和“暂时不能写”的边界上。

它不是用户必须看到的交付格式，也不是固定表格模板。它是内部规划检查，用来判断某一章或某一节是否已经具备进入 `/UPTW-write` 的条件。

章节功能本身如何划分，见 `chapter-function-bank.md`；本文件只处理“该章想写的判断有没有证据支撑，以及在什么推演模式下可以写”。

## 何时加载

- 用户刚完成 `/UPTW-plan`，准备锁定后续写作章节
- 某一章材料很多，但哪些内容可写、哪些内容仍是猜测还不清楚
- 章节主题已经确定，但图、表、公式、结论之间还没有形成稳定映射
- 需要判断“这一段能不能写”“这一章能不能开写”
- `/UPTW-write` 命中 blocker，必须回退 plan 修复

## 核心原则

- 先锁定章节功能，再锁定章节主张。
- 先锁定证据锚点，再锁定 `reasoning_mode`。
- 没有证据锚点的判断，不进入正式写作。
- 证据不足以支持当前推演强度时，退回 `open_question` 或 `do_not_write`。
- 可以保留开放问题，但开放问题不能伪装成已证成结论。

## 章节级最小对齐项

每个准备进入 `/UPTW-write` 的章节，至少要回答下面这些问题：

1. 这一章的功能是什么。
2. 这一章要回答的核心问题是什么。
3. 这一章允许调用哪些材料。
4. 这一章最重要的 1 到 3 个判断分别是什么。
5. 每个判断各自对应哪些证据锚点。
6. 每个判断允许使用哪种 `reasoning_mode`。
7. 哪些判断已经可以写，哪些仍然只能作为开放问题。
8. 哪些内容绝对不能超出材料边界自行补写。
9. 这一章依赖前文哪些输出，写完后又将给后文提供哪些输出。

## 可用的证据锚点

证据锚点应优先从用户材料中定位，可包括：

- 现有 DOCX 中已经写明并经过用户保留的事实
- 图件、表格、公式及其编号
- 数据来源说明、统计结果、模型输出
- 用户明确确认过的结论
- 开题报告、研究设计说明、变量定义、术语表
- 章节 brief 中已登记的 `sources`、`confirmed_facts`、`required_figures`、`required_formulas`

不要把“常识上看起来合理”“同类论文通常会这样写”当作证据锚点。

## 推荐的内部对齐格式

无需强制展示给用户，但内部可按下面的思路检查：

| 项目 | 要回答什么 |
| --- | --- |
| 章节功能 | 本章是在界定问题、说明方法、报告结果、解释机制，还是提出策略 |
| 核心判断 | 本章最重要的可检验判断是什么 |
| 证据锚点 | 这个判断对应哪些图、表、公式、数据、已确认结论 |
| 推演模式 | `describe` / `compare` / `correlate` / `interpret_cautiously` / `strategy_translate` |
| 状态 | `supported` / `open_question` / `do_not_write` |
| 依赖 | 这个判断依赖前文哪个 confirmed output |
| 风险 | 是否存在术语不稳、证据不足、边界不清、对象切换 |

## 按章节功能的对齐提醒

### 绪论或引言

- 允许写：研究对象、研究范围、研究问题、材料基础、论文结构、来自用户材料的现实背景
- 谨慎写：理论意义、现实意义、创新点，必须能回指已有研究设计或已完成成果
- 不要写：没有材料支撑的“显著填补空白”或泛化价值判断

### 研究区、数据与资料来源

- 重点锁定：空间单元、时间范围、数据来源、口径、尺度和变量
- 不要把数据名称写清了，却把采集边界和适用边界留空

### 概念、理论、文献与方法

- 每个概念、指标、变量、模型都要能对应后文用途
- 每个公式或计算步骤应能用自然语言叙述其推导逻辑和计算关系；如果某公式的变量含义尚不能用中文清晰解释，则该公式不算可写完成
- 如果某个方法名称已经出现，但输入、输出、参数、适用范围还不清楚，就不算可写完成

### 结果章节

- 每个重要判断都应能回到一个具体图、表、公式、统计结果或模型输出
- 如果只能写“呈现某种趋势”，却不能说明趋势对应的对象、尺度和数据边界，这一判断还不够稳
- 结果层判断默认从 `describe` 或 `compare` 起步，不自动升级到解释

### 机制、影响因素或解释章节

- 解释必须回扣前文结果，不是另起一套新话题
- 若只有相关性证据，禁止写成确定性因果
- 解释类判断通常只能使用 `correlate` 或 `interpret_cautiously`

### 策略章节

- 每条策略都要能回指前文诊断问题和证据来源
- 至少能回答：针对谁、针对什么问题、依据什么证据、用什么工具、边界在哪里
- 没有诊断映射的策略不进入正文

### 结论章节

- 只收束已经写明并有支撑的发现、贡献、限制和展望
- 不在结论里新增未展开论证的新判断

## 发现证据缺口时怎么处理

- 能通过现有材料补齐的，记录到 chapter brief 再进入写作。
- 暂时不能补齐的，作为开放问题保留。
- 影响全章成立的缺口，优先回到用户确认，而不是带着缺口开写。
- 若缺口会影响 downstream section，直接写入 `replan_queue.json`。

## 不推荐的做法

- 先想一句“像论文结论的话”，再去找材料倒贴
- 让一个章节承担过多功能，既想界定问题，又想报告结果，又想提出策略
- 把图表当装饰品，正文判断却不回指图表
- 因为同类论文通常会写某种内容，就默认用户这篇也能写
- 在 `/UPTW-write` 中临时升级推演强度，绕过 plan 阶段冻结的 `reasoning_mode`
