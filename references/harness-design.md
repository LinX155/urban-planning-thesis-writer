# 长文学位论文 Harness 设计原则

本文件用于说明网上中文学术写作提示词和长文协作经验对本 skill 的影响。它不是提示词库，也不提供可复制句式。

## 参考结论

- 常见中文“去 AI 味”材料多服务于段落润色、降重复感和去模板腔，不能替代数万字论文的状态管理、DOCX 协作和事实边界控制。
- 学术写作辅助必须限于组织、表达、衔接、检查和格式协作，不能替代选题、实验、模型分析、田野材料或结果判断。
- “自然化”应通过补足证据链、调整段落节奏、减少机械总结和避免空泛连接实现；不得故意打乱逻辑、弱化术语或插入口语化缺陷。
- 长文 harness 必须分阶段运行：初始化状态，计划章节和事实边界，再逐章写作和审阅，不应依靠一次性长提示词完成全文。
- 参考 OpenSpec 的 artifact-guided 思路：大型 harness 不能只靠一份主提示词，还应把关键决策落成项目内工件，减少对聊天上下文的依赖。
- 参考 OpenSpec 的 brownfield-first 思路：已有 DOCX、图表、实验材料和用户修改优先于新生成内容。

## 对本 skill 的约束

- plan/write 是用户可见阶段；质量审查、术语检查、图表公式检查、反 AI 模板腔、恢复和备份都是内部机制。
- 每次写入 DOCX 前后都应有快照、备份、差异摘要或日志，避免覆盖用户审定内容。
- 每个准备进入写作的章节都应有 chapter brief，每轮写作/审阅都应有 review cycle 工件。
- 不把外部提示词、论文语料或单篇亮点转化为固定句式库。
- 对网上提示词中涉及选题、实验设计、数据分析、代写、生成结论的部分一律不吸收。
