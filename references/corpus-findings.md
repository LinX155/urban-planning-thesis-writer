# 语料调研凝练结论

本文件从用户提供的城市规划硕士论文 PDF 中凝练写作共性、少数亮点和风险。它的职责是保存“从语料里观察到了什么”，而不是重复承担 plan、write 或最终行文规范的执行说明。语料抽取、子 agent 笔记和主 agent 完整读文审查属于创建过程文件，不放入最终 skill。

## 可进入 skill 的共同写作共性

- 章节通常承担明确功能：绪论提出问题，理论/文献界定概念和研究切入，方法章节说明数据、指标和模型，结果章节呈现实证发现，机制或影响因素章节解释原因，策略章节回应前文诊断，结论章节收束。
- 城市规划论文强调空间尺度、研究对象和指标体系的对应关系。写作时必须反复确认“研究对象是什么、空间单元是什么、时间范围是什么、数据支撑是什么”。
- 空间条件不是背景装饰。山地、山区、都市圈、省域、县域等条件应进入指标、分层、机制解释或策略对象。
- 研究对象需要可操作化。数字经济、生产空间、公共空间、空间正义等概念必须明确边界、单元和指标。
- 概念界定通常不止于定义，还要说明来源依据、空间/时间范围、行业或指标口径、排除对象和后文用途。
- 复杂概念需要分层：现象、机制、状态、绩效和代理指标不能混写。
- 尺度锁定是长文一致性的核心。城市、县域、城市群、格网、土地类型等口径必须稳定。
- 图表不是展示附件，而是论证证据。正文需要把图表嵌入分析链条：引入目的、读取方式、关键发现、后文用途。
- 公式通常服务于评价、测度、网络分析、模型或模拟。公式表达应与正文解释连续，不应把变量说明和结果解释留空。
- 方法密集论文尤其要求可复核：数据、指标、变量、模型、参数、评价口径和解释方法不得缺位。
- 方法不是术语清单。同一空间分析、评价或统计方法必须说明它在本文中回答什么问题，以及结果将如何进入下一章。
- 综合评价类论文通常需要说明模型在问题链中的角色：水平测度、系统关系、速度关系、空间异质性、类型划分或路径选择各不相同。
- 网络类论文尤其要求概念稳定：节点、边、权重、方向、阈值、时间窗口和数据边界必须先说明。
- 结果分析常从总体到局部、从时间到空间、从单维到多维、从现象到原因推进。具体顺序应由用户材料决定。
- 章节小结承担承上启下功能，应回到本章维度、指标、类型或空间单元，压缩出后续章节可调用的判断。
- 同一大主题内部仍需根据研究对象、区域尺度、数据来源和核心发现调整章节主轴，不得机械复用上一篇同类论文的框架。
- 策略建议不能脱离实证章节，必须与前文问题、差异、机制或评价结果一一对应。
- 策略章节前通常需要关键问题、发展趋势、耦合关系或类型诊断作为中介，不应从结果段直接跳入对策。
- 耦合协调、承载力、可持续城市化等主题应同时写单系统状态、系统关系和短板类型，避免总指数掩盖具体问题。
- 创新走廊和空间交通协同主题必须证明节点、轴线、要素流动、功能组织和治理关系；交通线或道路清单本身不足以支撑“走廊”或“协同”结论。
- 案例比较论文应先统一比较维度，再分案例展开，并在每一轮比较后回收同异和启示。
- 策略章节尤其需要可检查对象，不能把“优化、完善、强化、提升、协同、因地制宜”等词当作策略本身。
- 结论不新增未论证内容，常需区分主要发现、理论/方法贡献、规划启示、局限与展望。

## 可作为建议的少数亮点

- 宏观格局与微观过程结合，适合人口流动、城市网络、产业联系等主题。
- 多维度框架先分后合，适合用户材料确实包含多个独立评价维度的论文。
- 类型差异、阶段差异和机制差异联动分析，适合有分类或时序材料的论文。
- “问题-证据-策略”闭环能提升策略章节可信度。
- “诊断问题-证据来源-影响机制-策略对象-策略工具-预期作用-适用边界”映射表适合用作策略章节内部审查。
- “空间条件-指标诊断-类型模式-策略响应”适合区域综合评价和优化策略论文。
- “价值判断指标化”适合公平、正义、宜居、高质量等价值导向主题。
- “结构识别-因素解释-绩效检验”适合空间结构与绩效关系论文，但结论必须带研究边界。
- “模拟证据-空间识别-规划响应”适合生态模拟和国土空间分区类论文。
- “指标测度-空间格局-类型识别-因素解释-分类策略”适合耦合协调、承载力和可持续城市化等评价类论文。
- “节点-轴线-流动-治理”适合创新走廊、交通协同和跨界空间治理类论文，但必须由用户材料支撑。
- 在计划阶段建立术语表、变量表、图表公式清单，可显著降低长文前后不一致。
- “理论-维度-指标-模型-策略”与“数据-指标-模型-解释-策略”是可参考的链条，但只有在用户材料完整时使用。
- “数据建网-指标测度-类型划分-机制解释-优化建议”是网络论文的可参考链条，但必须绑定具体数据和研究尺度。
- 特殊群体、城市群网络、多尺度评价等框架只有在用户材料匹配时才可采用。

## 不应泛化或应主动规避

- 不把任何一篇论文的专属框架套用到用户主题，例如“等级-网络”“收缩效应”等只能在材料匹配时使用。
- 不因主题相近就复用理论组合、章节顺序或结论分点。
- 不凭空生成模型排序、机器学习阈值、参数、评价指标、绩效结论或尺度差异。
- 不把百度迁徙、搜索指数、列车班次、问卷调查等不同口径数据硬合成为单一结论。
- 不把相关性、回归方向或代理变量写成确定因果和完整概念。
- 不把情景模拟结果写成确定事实，也不让重复 PDF 重复计权。
- 不为追求专业感堆叠理论名词。理论只服务于研究对象和解释路径。
- 不追求图表数量。图表密度高但解释不足，会削弱论文说服力。
- 不形成高频表达句库。语料只提供结构原则，不提供可复制句子。
- 不用空泛价值判断替代论证，尤其避免没有对象、证据和边界的“意义”“参考”“启示”。
- 不生成无证据的四字短语、排比标题或规划口号；凝练概括必须来自用户材料并紧接证据解释。
- 不把“去 AI 味”理解为同义词替换、故意打乱逻辑或口语化。自然化只应调整论证衔接、句式节奏和机械总结，不得牺牲学术严谨性。
