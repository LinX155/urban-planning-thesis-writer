# 中文城市规划硕士论文行文约束

本文件用于 `/UPTW-plan` 与 `/UPTW-write`。它不是句式库，禁止复制语料论文原文或生成固定模板句。

关于“某种证据允许写到哪一步”的问题，不在本文件内部展开，请同时加载 `inference-boundaries.md`。本文件只负责写作层约束；是否可写、能写多强，由 `chapter-evidence-alignment.md` 与 `inference-boundaries.md` 共同决定。

## 总体原则

- 从用户已有材料出发：论文文字只能组织、解释和连接用户已经提供或确认的事实、数据、图表、公式、结论。
- 长文优先：保持章节层级、论证节奏和前后承接，不把数万字论文写成短报告式分点摘要。
- 证据优先：先交代研究对象、数据、指标、图表或模型，再推出判断；不要先给结论再补理由。
- 推演强度受限：正文不得超出当前 judgment 对应的 `reasoning_mode`。
- 研究对象先可操作化。价值词、产业词和空间词必须转为可检查对象、空间单元、数据来源或指标。
- 核心概念先锁边界。定义应同时说明来源依据、空间/时间范围、行业或指标口径、排除对象和后文用途。
- 专业克制：使用准确、稳定、可核查的学术汉语，避免夸饰、口号、宣传语和公众号式表达。
- 减少解释性比喻。除非用户材料或原始理论概念本身必须如此表述，不用比喻、拟人或形象化转述替代概念界定、机制解释和证据说明。
- 一致性优先：研究对象、空间尺度、时间范围、指标名称、变量缩写、图表编号和章节标题前后一致。
- 推导思路自然语言化：涉及计算或推导时，先用自然语言叙述推导逻辑和步骤关系，再以公式作为辅助。读者应能通过文字理解推导思路，公式只是对文字描述的精确化补充，不是替代。
- 不应为了”学术风格”而将语言表达刻意复杂化。通俗易懂与学术严谨并不矛盾，清晰直白的表述优于晦涩绕弯的表述。
- 反 AI 模板腔必须服务学术严谨。不得为了”自然”或”降 AI”故意打乱逻辑、降低术语精度、插入口语化缺陷或弱化证据。

## 绪论/引言

建议检查以下功能是否齐全，而不是强行套固定结构：

- 研究背景是否从宏观问题收束到具体研究对象。
- 研究问题是否明确，且与用户已有材料相匹配。
- 理论意义和现实意义是否具体指向研究对象、数据材料、方法组织或规划决策，避免空泛价值判断。
- 研究区域、数据来源、研究内容、技术路线是否与后文一致。
- 创新点必须来自真实材料和方法组织，不得为了完整性编造。

## 理论、文献与方法章节

- 先界定核心概念，再说明相关理论或研究进展如何服务本文。
- 概念界定应包括来源依据、对象边界、范围排除项和可观测指标。产业类别、空间类型、群体类型、政策概念不得只写词典式定义。
- 文献综述不是罗列作者和观点，应形成“已有研究做了什么、本文还需要解决什么”的过渡。
- 方法章节必须说明数据来源、指标含义、计算对象、适用范围、局限和结果用途。
- 同一方法在不同章节可能承担不同角色。写方法时必须说明它回答的问题，例如识别分布强度、方向变化、集聚类型、热点冷点、影响因素或策略分区。
- 公式前说明为什么需要该公式；公式后解释变量、单位、取值、计算对象和后文使用方式。
- 文内公式（嵌入正文行中的公式）出现时，对其中首次出现的变量以括号方式解释含义；换行公式（单独成行的公式）出现时，另起一行逐一说明式中每个变量的含义。
- 符号按需使用：如果某个计算关系用中文即可清晰表达且后文不再引用该符号，应优先使用中文描述，不必引入变量符号。变量符号只在表达复杂、需要精确指代或后文反复引用时才引入。
- 检查所有中文表达是否存在歧义。对于可能被多种方式理解的表述，应改写为只有一种理解方式的措辞。
- 当文中出现阈值时，应加入文字说明该阈值的含义、为何取此值、以及取此值意味着什么。不能只给出一个数值而不解释其判定逻辑。
- 模型出现前说明研究目的、适用对象、输入数据、输出结果和选择理由。
- 每个变量解释含义、单位、取值来源和正负向属性。
- 指标体系应形成“维度-指标-数据源-计算方式-作用方向”关系；缺项标为待补并询问用户。
- 评价类指标体系必须说明各维度对应的规划问题，不能只列指标。城镇化、承载力、可持续、协同、效率等综合概念都应拆成可解释维度。
- 机器学习结果必须绑定训练数据、特征变量、参数或调参方式、评价指标和解释方法。材料缺失时只能写“拟采用”，不能写“结果表明”。
- 尺度效应必须说明研究范围、网格粒度、时间段、对比对象等口径。
- 城市网络类研究必须定义节点、边、权重、方向、阈值、时间窗口和数据来源。
- 多源流数据必须说明采集口径、时间尺度、行为含义和可比性边界。
- 复杂概念需建立“核心概念-观测指标-理论角色-模型变量”对应关系。
- 代理变量必须标明边界，不得把专利数量、DEA 效率、人口变化等直接等同于完整概念。
- 如果用户未提供方法细节或变量定义，必须询问，不得补写。

## 结果与机制分析

- 写作前先判定研究对象、区域尺度、数据来源和核心发现。主题相近不等于结构相同。
- 空间条件必须进入研究设计和结果解释。山区、山地、都市圈、省域、县域等标签应转化为指标、尺度、空间单元或机制变量。
- 空间单元必须与数据和问题匹配，并说明分析单元、选择理由和数据粒度一致性。
- 尺度必须前后一致。城市体系、县域、城市群、城市、格网、土地利用类型等口径不能混用。
- 按“总体特征-分维度/分类型/分阶段特征-差异解释”推进，具体顺序服从用户材料。
- 空间格局类结果要说明尺度、方向、集聚/扩散、核心-边缘、节点差异或区域差异。
- 时序演化类结果要说明阶段划分依据、变化趋势、拐点或稳定性。
- 模拟/情景结果要说明情景名称、时间点、约束规则、模型输入、参数来源和适用边界，不得写成确定事实。
- 评价类结果不能止于排名，要解释高低原因、维度滞后、空间分异和时间演变。
- 耦合协调、弹性、脱钩、承载力、可持续状态等评价结果不能只写总指数。应同时解释单系统变化、系统关系变化、短板维度和类型差异。
- 创新走廊、交通协同等空间载体不能只等同于线状通道。应证明节点、轴线、要素流动、功能组织和治理关系。
- 案例比较类论文必须先统一比较维度，再分别叙述案例，最后提炼同异与可借鉴经验；避免材料并置后缺少归纳。
- 影响因素和机制分析必须回到前文结果，不得脱离实证发现泛泛解释。
- 章节小结应回到本章指标、维度、类型或空间单元，提炼可供下一章调用的判断；不要写成通用结尾。
- 因果措辞按证据等级使用。相关分析用“相关、呈现关联、可能反映”；回归分析用“影响方向、解释关系、模型结果显示”；没有准实验、工具变量、DID 等识别设计时，不得写“决定、必然导致、直接造成、证明因果”。
- 当 plan 已将某个 judgment 标记为 `open_question` 或 `do_not_write` 时，write 阶段不得把它改写为既成事实。
- 策略章节前应先组织关键问题、发展趋势、耦合关系或类型诊断，再进入规划响应；不要从结果描述直接跳到对策口号。
- 策略或规划对策要与前文诊断建立映射。建议内部检查链为：诊断问题 -> 证据来源 -> 影响机制 -> 策略对象 -> 策略工具 -> 预期作用 -> 适用边界。
- 从评价或模型结果走向策略时，至少建立“诊断类型-证据指标-空间单元-策略动作”的映射。
- 交通类策略应说明交通网络如何服务空间结构调整、功能疏解、要素流动或区域协同，不能写成单独的道路/轨道建设清单。
- 优化建议必须回指具体问题、指标或空间类型。
- 生态价值转译要克制。碳储量、生态增汇、土地利用优化等只能支撑相应空间管控和要素导向，不能无依据推出泛化产业建议。
- 凡出现“优化、完善、强化、提升、促进、保障、协同、因地制宜”等词，必须回答：优化什么指标，在哪个空间单元，依据哪条诊断，采用什么工具。
- 凡出现“高质量、协同、公平、正义、宜居、产业升级、数字经济”等价值或产业词，必须落到对象边界、指标或证据。

## 图、表与公式

- 图表出现前：说明为何需要该图表、它回答哪个问题。
- 图表出现后：解释关键格局、差异、数值关系、异常或对后文的启示。
- 表格不能只复述表题，要说明指标分组、读表方式和关键变化。
- 地图、网络图、热力图等空间图件要解释空间单元、图例含义和空间关系。
- 公式不能孤立出现；必须与指标体系、模型、测度目标或结果解释相连接。公式前后应有自然语言的推导叙述，让读者理解公式的计算逻辑和推导步骤，而非仅看到符号运算。
- 网络指标、机器学习指标、评价指标和政策评价工具都要解释“它回答什么问题”，不能只列方法名。
- 中心性、密度、凝聚子群、社群、联系强度等网络指标只能说明网络关系位置或结构，不能直接替代城市综合实力判断。
- 网络图前说明展示目的；图后解释空间差异、关键节点/边、可能机制或规划含义。
- 方法章节不是术语展览，结果章节不是模型光环；所有判断都应回到数据、图表和可复核口径。
- 复杂结论需绑定研究区、年份、数据口径、模型方法和指标代理。
- 更新 markdown 时维护图表和公式清单，检查编号、题注和正文引用是否一致。

## 反 AI 模板腔

本节只保留硬边界；具体坏模式、典型症状和改写顺序见 `anti-template-patterns.md`。

避免：

- 过度分点、空泛过渡、机械总结和换词复述共同出现，把章节写成模板化信息拼贴。
- 用“具有重要意义”“提供参考”“奠定基础”等姿态句替代对象、尺度、指标和证据说明，口语化或公众号式表达。
- 只做同义词替换、句式打散或语气软化，却不修复论证链条、证据锚点和段落任务，过度使用修辞、说废话与重复表述相同内容。
- 为了“去 AI 味”而使用不严谨、不准确或过度文学化的表达。

允许：

- 在结论、策略、研究内容等确需并列的地方使用分点，但是必须是段落式分点，依然不要过度总结凝练。
- 在章节开头用短段说明本节任务，但必须具体到研究对象、材料和问题。
- 使用自然的承接句，但承接句应推动论证，而不是填充篇幅。
- 在不改变事实和术语的前提下调整句式节奏，减少机械总结和重复连接。

## 用户修改记忆

- 将用户审阅后的修改作为证据，而不是命令。
- 只在修改体现稳定术语、结构原则、学术表达标准、数据边界或导师要求时记录为偏好。
- 对一次性的句子润色、局部事实补充、删除冗余内容，不自动泛化。
- 任何会影响后续全文写法但理由不明的修改，都应询问用户。
